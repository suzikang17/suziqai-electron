# Test Library Design Spec

**Date:** 2026-03-27
**Status:** Approved

## Summary

Add a file-system-first test library to suziQai so users can save tests from the session and browse/load/delete them from a library view in the sidebar. Saved tests are Playwright `.spec.ts` files with `.suziqai.json` sidecar metadata files, stored in the project's `testOutputDir`.

## Goals

- Users can explicitly save tests from the authoring session to the library
- Users can browse saved tests in a library view within the existing sidebar
- Users can load saved tests back into the session for editing/replay
- Users can delete saved tests
- Saved tests are valid Playwright spec files, immediately runnable with `npx playwright test`
- Future: import existing Playwright tests, suite groupings via subdirectories

## Architecture

### File-System-First Approach

The test library is the file system. No database or index file. The main process scans `testOutputDir` to list saved tests, reads sidecar files for metadata, and writes both `.spec.ts` + `.suziqai.json` on save.

```
<testOutputDir>/
├── login-flow.spec.ts          # Valid Playwright test
├── login-flow.suziqai.json     # suziQai metadata sidecar
├── checkout-happy-path.spec.ts
├── checkout-happy-path.suziqai.json
└── legacy-test.spec.ts         # Existing test, no sidecar → shows as "imported"
```

### Data Flow

**Saving a test:**
1. User clicks "Save" on a test in session view
2. Renderer sends `library:save` IPC with the `TestCase` data
3. Main process generates `.spec.ts` using existing spec generation
4. Main process writes `.suziqai.json` sidecar alongside it
5. If the test was loaded from the library, overwrites the same files
6. If new, derives filename from test name (slugified), appends numeric suffix on collision

**Loading a test:**
1. User clicks a test in library view
2. Renderer sends `library:load` with the fileName
3. Main process reads the `.suziqai.json` sidecar, returns full `TestCase`
4. Test is added to the current session's test tabs
5. Sidebar switches to Session mode automatically

**Listing saved tests:**
1. Main process scans `testOutputDir` for `*.suziqai.json` files
2. Also finds `*.spec.ts` files without sidecars (marked as "imported")
3. Returns lightweight `LibraryEntry[]` list

**Deleting a saved test:**
1. Removes both `.spec.ts` and `.suziqai.json` files

## File Format

### Sidecar `.suziqai.json`

```json
{
  "id": "abc-123",
  "name": "Login flow",
  "steps": [
    {
      "id": "step-1",
      "label": "Navigate to login page",
      "action": { "type": "navigate", "url": "http://localhost:3000/login" },
      "status": "passed"
    }
  ],
  "savedAt": "2026-03-27T10:00:00Z",
  "updatedAt": "2026-03-27T10:00:00Z"
}
```

### Spec `.spec.ts`

Standard Playwright test file generated by existing spec generation logic. Valid and runnable as-is.

## IPC Channels

| Channel | Direction | Payload | Returns |
|---------|-----------|---------|---------|
| `library:list` | Renderer → Main | none | `LibraryEntry[]` |
| `library:save` | Renderer → Main | `{ test: TestCase, fileName?: string }` | `{ fileName: string, path: string }` |
| `library:load` | Renderer → Main | `{ fileName: string }` | `TestCase` |
| `library:delete` | Renderer → Main | `{ fileName: string }` | `void` |

## Types

```typescript
interface LibraryEntry {
  fileName: string;      // e.g., "login-flow" (no extension)
  name: string;          // Display name from sidecar, or fileName for imported
  stepCount: number;
  savedAt: string;       // ISO date
  updatedAt: string;     // ISO date
  imported: boolean;     // true if no sidecar exists
}
```

## UI Changes

### Sidebar Dual-Mode Toggle

A tab bar at the top of StepSidebar: **Session** | **Library**

- **Session mode:** Exactly the current behavior (test tabs, step list, Run All, Export)
- **Library mode:** Scrollable list of saved tests from `testOutputDir`

### Library View

- Each entry shows: test name, step count, last updated date
- Clicking a test loads it into the session (auto-switches to Session mode)
- Delete button with confirmation on each entry
- Tests without sidecars show an "imported" badge

### Save Button

- Added to the test tab header in Session mode
- If test was loaded from library, overwrites the same files
- If new, prompts for a name (pre-filled from test name)
- Brief visual confirmation on successful save

### No Other UI Changes

Browser viewport, toolbar, and chat panel are unchanged.

## Error Handling

- Save to read-only directory → surface error message in UI
- Corrupt/missing sidecar fields → treat as imported test (graceful degradation)
- File name collision on save → append numeric suffix (`login-flow-2.spec.ts`)

## Testing Strategy

- Unit tests for `test-library.ts` (save/load/list/delete with temp directories)
- Component test for `LibraryView.tsx` (renders entries, handles clicks)
- Update existing `StepSidebar.test.tsx` for toggle behavior

## Future Extensibility

- **Suites:** Subdirectories in `testOutputDir` become suite groupings
- **Import Playwright tests:** `.spec.ts` without sidecars already appear in library. Future work parses them into `TestCase` steps.
- **Tags/search:** Add fields to sidecar JSON, filter in LibraryView
